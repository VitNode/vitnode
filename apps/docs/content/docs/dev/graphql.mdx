VitNode uses [GraphQL](https://graphql.org/) to query and mutate data with [Codegen](https://the-guild.dev/graphql/codegen) to generate TypeScript types. It's a powerful tool that allows you to query only the data you need and get the data in the shape you want.

## GraphQL Playground

VitNode comes with a built-in GraphQL Playground powered by [Apollo Sandbox](https://www.apollographql.com/docs/graphos/explorer/sandbox/). You can access it by going to [localhost:8080/graphql](http://localhost:8080/graphql) on your website.

## Folder for Feature

To save consistency and make it easier to maintain, we're going to create a module to store all the queries and mutations. Each sub-module should have its own module.

For example if you have `blogs_categories` and `blogs_articles` in the database, you should have two modules, one for each feature, like `BlogCategoriesModule` and `BlogArticlesModule`.
